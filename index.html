Tool Overview
✅ Compliance Summary: This tool retrieves campaign performance data from a single Google Ads account via read-only API access. It does not create, modify, or delete any campaigns, bids, or budgets. All data remains local and is never shared externally. Access is strictly limited to the developer — no multi-user functionality or external access is provided.
What This Tool Does

This dashboard connects to the Google Ads API using OAuth 2.0 to retrieve campaign performance metrics from a Google Ads account managed by Wenlong Du. Retrieved data is displayed in a private local dashboard for analysis and campaign monitoring.

The tool is designed exclusively for reporting and analysis. It does not perform any write operations or automated actions on the connected account.

✅ Capabilities
  • Retrieve campaign metrics via Google Ads API
  • Display impressions, clicks, cost, CTR, conversions
  • Break down performance by campaign, ad group, keyword
  • Track spend and budget data over selected date ranges
  • Query data using GAQL (read-only)
🚫 Out of Scope
  • No campaign creation or modification
  • No bid or budget changes
  • No access to third-party accounts
  • No external data sharing or transfer
  • No automated actions of any kind

All API requests are read-only. This tool does not create, modify, pause, or delete any campaigns, ads, bids, or budgets. It queries performance data via GAQL from a Google Ads account owned and operated by Wenlong Du.

GAQL Query — Read-Only Reporting
SELECT
  campaign.id,
  campaign.name,
  campaign.status,
  metrics.impressions,
  metrics.clicks,
  metrics.ctr,
  metrics.cost_micros,
  metrics.conversions,
  metrics.cost_per_conversion
FROM campaign
WHERE segments.date DURING LAST_30_DAYS
ORDER BY metrics.cost_micros DESC

Authorization Flow
Access to Google Ads data is authorized through the official Google OAuth 2.0 flow using the https://www.googleapis.com/auth/adwords scope. Only the Google Ads account owned and managed by Wenlong Du is connected to this tool.

Data Access & Authorization
How This Tool Accesses Google Ads Data

This tool accesses Google Ads campaign data through the official OAuth 2.0 authorization flow. The developer authorizes the tool to connect to a specific Google Ads account using the https://www.googleapis.com/auth/adwords scope. Once authorized, the tool retrieves campaign performance metrics from that account for local reporting and analysis.

All API requests are read-only GAQL queries. No write operations of any kind are performed. Retrieved data is stored locally and is never transmitted to external services or third parties.

Authorization Steps
  • Developer initiates OAuth 2.0 flow via Google
  • Google prompts authorization for the adwords scope
  • Developer grants access to their own Google Ads account
  • Access token stored server-side for API requests
  • Dashboard retrieves and displays campaign data
Why API Access Is Required
  • Google Ads campaign data is only accessible via the official API
  • OAuth 2.0 is the required authentication method for API access
  • The adwords scope is needed to query campaign metrics
  • GAQL queries require an authorized API session to execute
Access Restrictions

Access to this tool is strictly limited to the developer, Wenlong Du. The tool provides no multi-user functionality, no public login system, and no mechanism for external parties to connect or authorize their accounts. Only the developer's own Google Ads account is connected via OAuth 2.0.
